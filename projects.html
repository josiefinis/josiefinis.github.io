Lanterns

<> Code

I have a number of LED tea light candles and wanted a way to turn them on and off all together, using any one of them as the switch. My idea was to use capacitive touch sensing so that each light can be controlled individually with a simple touch. With a longer touch the neighbouring lights should also turn on and then their neighbours and so on, like a chain of signal beacons lighting up to warn of an invading armada. I also wanted the lights to turn off automatically at slightly different random times around four hours later, as though they were real candles.

I used an Arduino UNO controller and made two iterations. The first was very simple and minimalistic. I wanted to prototype a design that could be implemented with a simple electronic circuit. This iteration was a simple state machine where the state of each light, a counter, and a couple of flags were stored in a single byte.

For the second iteration I wanted to be able to vary the brightness of each light. I wanted each light to slowly dim over time before going out and for there to be some variation in brightness between the diffent lights. I feel this gives a more naturalistic and aesthetically pleasing appearance. I also wanted to be able to control the brightness of each light individually and all of them as a group. This presented some interesting technical and UX challenges given the limitations of the hardware.

Visual Odometry

[pdf] Report

This project was for a course in Applied Estimation as part of the Systems Control and Robotics program at KTH. A significant part of this course was on using the Kalman filter to estimate the position of a robot within a map by combining odometry and measurements to known landmarks. I used video input from a pair of cameras on the robot to generate both the odometry and to detect and locate landmarks relative to the robot.
